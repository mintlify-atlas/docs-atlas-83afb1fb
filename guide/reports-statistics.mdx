## Generating Historical Reports

The **Historial de Solicitudes** module provides comprehensive historical reporting.

### Accessing Historical Data

<Steps>
  <Step title="Navigate to History Module">
    From the main menu, go to **Módulo de Solicitudes** → **Historial de Solicitudes**.
  </Step>

  <Step title="View All Requests">
    The system displays all historical requests with the following statuses:
    - INICIO
    - PROCESO
    - EN PREVENCIÓN
    - EN AMPLIACIÓN
    - TERMINADA
    - DESECHADA
    - CANCELADA
  </Step>

  <Step title="Apply Filters">
    Use the filter controls to narrow down results:
    
    **Text Search:**
    - Search by requester name (Nombre)
    - Search by folio number
    - Real-time filtering as you type
    
    **Status Filter:**
    - Select specific status from dropdown
    - View all statuses or focus on one
    
    **Date Filters:**
    - **Specific Day**: Select exact date
    - **Month**: Choose any month
    - **Year**: Filter by year
  </Step>

  <Step title="Review Results">
    The filtered table shows:
    - ID and Folio
    - Requester name
    - Start date
    - Current status (color-coded badge)
    - **Ver Detalles** button for each request
  </Step>
</Steps>

### Report Pagination

Large datasets are paginated for better performance:

- **Default page size**: 12 requests per page (responsive)
- **Navigation**: Previous/Next buttons
- **Current page indicator**: "Página X de Y"
- **Automatic page size**: Adjusts based on screen resolution

## Downloading Excel Reports

Export complete request data to Excel for advanced analysis.

<Steps>
  <Step title="Set Your Filters">
    Configure the date and status filters to specify which requests to include in the export.
  </Step>

  <Step title="Click Download Excel">
    Click the **📤 Descargar Excel** button in the top right.
  </Step>

  <Step title="Automatic File Generation">
    The system generates an Excel file with a descriptive name:
    
    ```plaintext
    Format: Reporte_{STATUS}_{DATE}_{TIME}.xlsx
    
    Examples:
    Reporte_TERMINADA_Marzo-2026_14꞉30꞉15.xlsx
    Reporte_TODOS_LOS_ESTADOS_11-03-2026_09꞉15꞉30.xlsx
    Reporte_EN_PROCESO_2026_16꞉45꞉00.xlsx
    ```
  </Step>

  <Step title="Open and Analyze">
    The Excel file includes comprehensive data (37 columns total):
    
    Open with Microsoft Excel, Google Sheets, or LibreOffice Calc.
  </Step>
</Steps>

## Appeal Resource Statistics

The **Recursos de Revisión** module provides statistical analysis of appeals.

### Generating Statistical Reports

<Steps>
  <Step title="Access Statistics Module">
    Navigate to **Recursos de Revisión** → **Estadísticas**.
  </Step>

  <Step title="Configure Filters">
    Select your analysis criteria:
    
    **Month Filter:**
    - Dropdown with all 12 months
    - Leave blank for all months
    
    **Year Filter:**
    - Text input for year (e.g., "2026")
    - Leave blank for all years
    
    **Subject Matter Filter:**
    - **DAI**: Access to information
    - **DP**: Personal data protection
    - Leave blank for both
  </Step>

  <Step title="Execute Query">
    Click the **Buscar** button to generate statistics.
  </Step>

  <Step title="Review Results">
    The system displays a table with:
    
    - **Concepto**: Statistical concept/category
    - **Cantidad de recursos de revisión**: Number of appeals
    - **Desglose de los Recursos de Revisión**: Detailed breakdown of resource numbers
  </Step>
</Steps>

### Statistical Concepts

The statistics report includes various analytical categories based on:

- Resolution outcomes (Confirma, Revoca, Modifica, Sobresee)
- Matter type (DAI vs DP)
- Time period analysis
- Subject categorization

<Note>
  Statistics are calculated based on **Fecha de Acuerdo Final** (final agreement date), not the initial filing date.
</Note>

### Clearing Filters

To reset all filters and start fresh:

1. Click the **Limpiar** button
2. All filter fields reset to empty
3. Statistics table clears
4. Ready for new query

## Advanced Excel Analysis

Once you have exported data, you can perform advanced analysis:

### Key Metrics to Calculate

<Steps>
  <Step title="Response Rate">
    ```plaintext
    = TERMINADA requests / Total requests
    Shows completion percentage
    ```
  </Step>

  <Step title="Average Response Time">
    ```plaintext
    = AVERAGE(Promedio Días column)
    Overall performance metric
    ```
  </Step>

  <Step title="Prevention Rate">
    ```plaintext
    = (Prevención = Sí) / Total requests
    Indicates request clarity issues
    ```
  </Step>

  <Step title="Appeal Rate">
    ```plaintext
    = (Recurso = Sí) / TERMINADA requests
    Shows satisfaction with responses
    ```
  </Step>
</Steps>

## Performance Considerations

<Warning>
  Large exports (thousands of records) may take time. Be patient while the system generates the Excel file.
</Warning>

Optimization tips:

- Use date filters to limit result sets
- Export specific statuses rather than all
- Schedule large reports during off-peak hours
- Consider monthly incremental exports vs. one annual export
